Java
  1. 开启GC日志,使用-XX:+PrintGCDetails -XX:+PrintGCDateStamps -Xloggc:/export/log/risk_pillar/gc.log
  2. 设置JVM垃圾最大年龄 -XX:MaxTenuringThreshold=15
  3. 打印所有命令参数 -XX:+PrintCommandLineFlags -version
  4. 使用压缩指针-XX:+UseCompressedClassPointers。64位虚拟机中在堆内存小于32GB的情况下，UseCompressedOops是默认开启的，该参数表示开启指针压缩，会将原来64位的指针压缩为32位。所以，在内存大于32G但又不是特别大的时候(比如36G-48G)，通过-xmx=31G限制最大堆内存，强迫JVM开启压缩指针。
