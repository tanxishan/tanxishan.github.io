TDD及单元测试
Posted on: 2024-07-31 Edited on: 2024-07-31 In: 编程思想
Java C#

单元测试遵循了叫做“准备/动作/断言(A/A/A)”的模式。

  • 准备（Arrange）：建立测试环境
  • 动作（Act）：执行测试
  • 断言（Assert）：判断能否得到所需要的某个结果

让单元测试有一致的结构，会使它们易于阅读。当项目含有数百个单元测试时，更会体会到这么做的好处！
